默认情况下, Write Cursor在前进的过程中, 只要环绕了 stripe 一次以上, 就会覆盖缓存(实际上是逐出缓存). 一些情况下这是不能接受的, 这个时候只能把对象从缓存中读取出来, 并写回到缓存中, 把 对象从物理存储中, 从Write Cursor前面写到了Write Cursor后面. 这会基于, stripe 中的数据结构进行处理, 从 而撤出缓存.

Evacuation data structures are defined by dividing up the volume content in to a disjoint and contiguous set of regions of EVACUATION_BUCKET_SIZE bytes. The Vol::evacuate member is an array with an element for each evacuation region. Each element is a doubly linked list of EvacuationBlock instances. Each instance contains a Dir that specifies the fragment to evacuate. It is assumed that an evacuation block is placed in the evacuation bucket (array element) that corresponds to the evacuation region in which the fragment is located although no ordering per bucket is enforced in the linked list (this sorting is handled during evacuation). Objects are evacuated by specifying the first or earliest fragment in the evactuation block. The evactuation operation will then continue the evacuation for subsequent fragments in the object by adding those fragments in evacuation blocks. Note that the actual evacuation of those fragments is delayed until the write cursor reaches the fragments, it is not ncessarily done at the time the first / earliest fragment is evacuated.

有两种清空的方式, 基于 reader 和 forced. EvacuationBlock 有一个 reader 计数来跟踪这个. 如果reader count是 0, 这是一个强制的性的 evacuation, 如果存在, 会在 write cursor 关闭的时候, 执行 evacuation. 如果 reader count不是 0, 这个值统计着当前能够读取对象的实体的数量. 当 readers 试着去访问对象的时候, 会增加 reader count. 或者, 创建 EvacuationBlock 的时候, 会初始化为 1. 当一个 reader 结束访问对象, 会减小这个值. 当reader count为 0 的时候, 会删除 EvacuationBlock . 如果, EvacuationBlock 以 0 的状态存在, 并且数量没有修改, reader 的数量也没有修改并且 reader 没有被跟踪. EvacuationBlock 会随着对象一真存在.
